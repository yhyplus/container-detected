# 项目文件清理建议

本文档用于说明项目中哪些文件是数据集生成过程的一部分，哪些文件可以考虑清理。

## 1. 原始采集表在哪里？

PPT 中提到的这些文件：

- `process_stats.csv`
- `network_stats.csv`
- `http_events.csv`
- `dns_events.csv`
- `sensor_events.csv`

不是最终训练数据集的列文件，而是每次实验运行目录下的原始采集表。

典型位置：

```text
runs/<run-name>/raw/<scenario>/process_stats.csv
runs/<run-name>/raw/<scenario>/network_stats.csv
runs/<run-name>/raw/<scenario>/http_events.csv
runs/<run-name>/raw/<scenario>/dns_events.csv
runs/<run-name>/raw/<scenario>/sensor_events.csv
```

例如当前答辩主要使用的新数据在：

```text
runs/train-v3-large-20260603/raw/normal/process_stats.csv
runs/train-v3-large-20260603/raw/http_flood/http_events.csv
runs/train-v3-large-20260603/raw/dns_flood/dns_events.csv
runs/train-v3-large-20260603/raw/portscan/sensor_events.csv
runs/eval-v3-large-20260603/raw/syscall_anomaly/sensor_events.csv
```

最终训练集是这些原始表聚合后的结果：

```text
runs/train-v3-combined-20260603/training_multiclass_windows.csv
runs/eval-v3-combined-20260603/training_multiclass_windows.csv
```

所以在 `result.csv` 或 `training_multiclass_windows.csv` 中看不到 `process_stats.csv` 这类文件名是正常的。

## 4. 可以考虑删除的文件

以下内容不影响当前答辩主线，可以考虑删除或移动到 `archive/`。

### Python 缓存

```text
__pycache__/
```

原因：Python 自动生成的缓存文件，可随时重新生成。

### UI 日志

```text
ui-logs/
```

原因：Web Console 运行日志，只用于排错。当前目录很小，但不影响答辩和训练。

## 5. 推荐清理策略

为了稳妥，建议分两步：

1. 先归档，不直接删除。

```text
runs/archive/
models/archive/
```

2. 答辩确认无误后，再删除归档中的旧数据。

如果只想让目录立刻清爽，同时不影响当前答辩，最小保留集是：

```text
源代码文件
k8s/
pxl/
templates/
static/
docs/
models/anomaly_detector.*
models/anomaly_binary_detector.*
models/anomaly_mlp.*
models/anomaly_mlp_binary.*
runs/train-v3-expanded-20260603/
runs/eval-v3-expanded-20260603/
```

## 6. 一句话解释给答辩用

原始表位于每次实验 run 的 `raw/` 目录中，最终训练数据集不是直接存这些表，而是把它们按 Pod 和 10 秒时间窗口聚合成 `training_multiclass_windows.csv`。因此 PPT 中提到的是数据集生成过程的原始输入，模型训练使用的是聚合后的窗口级特征表。
