# 重训练策略

本策略用于说明何时触发新一轮数据采集和模型重训练。

## 1. 触发条件

建议在以下情况触发重训练：

- 新采集的有效窗口样本达到 300 条以上。
- 某类异常样本明显不足，需要补齐类别分布。
- 漂移检测报告中的整体漂移分数持续升高。
- 在线告警数量突然异常增加。
- 新评估集上的 Accuracy 或 Macro F1 明显下降。
- 新增异常场景或服务类型。

## 2. 重训练流程

```text
采集新 train run
  -> 采集新 eval run
  -> 运行 mlops_train_pipeline.py
  -> 查看 MLflow 指标
  -> 对比旧模型
  -> 指标达标后替换默认模型
```

## 3. 模型替换规则

新模型需要满足：

- 六分类 Accuracy 不低于旧模型。
- Macro F1 不低于旧模型。
- 推理延迟没有明显恶化。
- 正常类误报率可接受。

## 4. 回滚策略

每次训练都会在 `registry/models/<version>/` 保存模型快照。如果新模型表现异常，可以将旧版本 artifact 复制回 `models/` 目录完成回滚。
